# @syntropic137/dashboard

Terminal dashboard for the design system. Launches Storybooks, runs repo tasks,
scaffolds the app-owned contract swap module, and explains the design-token standard.

## Usage

```sh
pnpm tui
```

Run from anywhere inside the monorepo — no arguments required.

## Context-aware menus

The dashboard detects where it is running and adjusts the menu accordingly.

**Inside the monorepo** (`pnpm-workspace.yaml` + `packages/contracts/` found up-tree):

```
? What would you like to do?
  Start a Storybook
  Run a repo task
  Install contracts into an app
  Design tokens standard
  Quit
```

**Outside the monorepo** (external app directory):

```
? What would you like to do?
  Install contracts adapter
  Set up design tokens
  Quit
```

## Install contracts — generating the swap module

"Install contracts into an app" is the flagship action. It:

1. Prompts for the target app directory and the implementation library (React 18 today;
   Svelte 5 once `rcl-q64.2` lands).
2. Reads the required and planned component names from the target's installed
   `@syntropic137/contracts` package (falls back to the known required set if not yet
   installed).
3. Prints the `pnpm add` / `npm install` command you need to run — does **not** auto-run it.
4. Writes `src/ui/adapter.ts` (or the path you specify) — idempotently; asks before
   overwriting an existing file.

The generated module looks like:

```ts
// Generated by @syntropic137/dashboard. The single swap point for this app's UI.
import type { RequiredComponentContracts } from "@syntropic137/contracts";
import { reactV18ContractAdapter } from "@syntropic137/default-react-v18";
// Swap implementations by changing this one import:
// import { svelteV5ContractAdapter } from "@syntropic137/default-svelte-v5";

export const ui = reactV18ContractAdapter satisfies Record<
  keyof RequiredComponentContracts,
  unknown
>;
```

To switch UI frameworks, change the single import — everything else in your app stays
the same.

## Design tokens explainer

"Design tokens standard" prints an inline primer covering:

- Semantic `--ds-*` CSS custom properties (consume, never hardcode values).
- Importing the layered CSS: `import "@syntropic137/design-tokens/generated/design-tokens.css"`.
- Theme swapping via `data-theme` on a root element.
- A sample of real `--ds-*` names read from the installed token CSS (in-repo only).

## Runtime note — no build step

This package runs via Node 22 native TypeScript type-stripping:

```json
"tui": "node --experimental-strip-types --disable-warning=ExperimentalWarning packages/dev-tools/dashboard/src/cli.ts"
```

Because there is no compilation step, intra-package imports use explicit `.ts` extensions
(e.g. `import { detectContext } from "./context.ts"`). This is a deliberate divergence
from the repo's `.js`-extension build convention and is confined to this package only.
