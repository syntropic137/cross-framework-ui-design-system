A Tauri window renders one web framework at a time. The contract and both adapters already exist and are type-safe — so the open question isn't whether you can swap implementations, it's what the swap gesture is and how much you build to get it. Three routes:

CONTRACT BUS RequiredComponentContracts → badgebuttontoggle ● reactV18ContractAdapter ● svelteV5ContractAdapter
Both adapters already implemented & conformance-enforced at typecheck (rcl-q64.1 / q64.2). The type-safe swap is done — these options differ only in the runtime story.
THREE ROUTES
RECOMMENDED
OPTION 1

Two builds,
one selector

ACHIEVABLE NOW · no new lib work
The swap looks like
# flip one selector + rebuild VITE_UI_LIB=svelte pnpm build
What you get
  • Real running demo: see React vs Svelte render the same contract UI
  • Proves parity end-to-end in the actual Tauri shell
  • Uses the adapters as-is
Cost
  • Harness carries both framework runtimes + two mount entries
  • “Swap” = rebuild, not live toggle
EFFORT
~½–1 day
Pick if you want to see it swap soon with what's already built.
OPTION 2

Web components
(custom elements)

TRUEST SWAP · bigger build
The swap looks like
<!-- framework-agnostic markup --> <ds-button>Go</ds-button> // swap = load a different bundle
What you get
  • Genuine one-import swap, any host (vanilla, React, Svelte, Tauri)
  • The cleanest "import into any app" end-state
  • Design truly decoupled from framework
Cost
  • New custom-element build target for both libraries
  • Slots/events/SSR & styling edge cases to solve
  • Really its own epic, not one bead
EFFORT
~1–2 wks
Pick if the real goal is drop into ANY app, and you'll invest.
OPTION 3

Type-level
swap only

LIGHTEST · defers runtime
The swap looks like
// src/ui/adapter.ts — one line import { reactV18ContractAdapter } from "@design-system/react-v18";
What you get
  • Proves the type contract + conformance now
  • Documents the swap point; near-zero work
  • Harness stays single-framework (React)
Cost
  • No running Svelte render — nothing visual to "see"
  • Defers the actual demo to later
EFFORT
~1 hr
Pick if you only need to prove type-safety and move on.

Decide in two questions

1 · Do you need to SEE it swap at runtime, or just prove the contract?
   see it → Opt 1 or Opt 2 · prove only → Opt 3

2 · Is the goal "swap inside this Tauri app" or "drop into any app, anywhere"?
   this app → Opt 1 · anywhere → Opt 2 (epic)

My read

Opt 1 now → it's the demo your end-goal asks for, built from what already exists. Opt 2 later, as its own epic, when "any app" matters. ↳ Opt 3 only if you want to keep moving.
